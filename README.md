# Тестирование мобильного приложения метро

## О проекте

В рамках проекта проводилось функциональное и регрессионное тестирование мобильного Android-приложения.

Основная цель:
- проверить функциональность после рефакторинга;
- провести регрессионное тестирование;
- выявить дефекты перед публикацией новой версии приложения.

Тестирование выполнялось на Android-эмуляторе в Android Studio.

---

## Что было сделано

### Анализ требований
- выполнен анализ требований, затронутых изменениями;
- подготовлен функциональный чек-лист;
- определены сценарии для проверки изменённой функциональности.

### Регрессионное тестирование
Подготовлен регрессионный чек-лист с учётом:
- работы сети;
- переключения между приложениями;
- изменения ориентации экрана;
- работы уведомлений;
- поведения приложения при низком заряде батареи;
- работы приложения при изменении состояния устройства.

### Тестирование приложения
Проведено:
- функциональное тестирование;
- регрессионное тестирование;
- проверка мобильных сценариев;
- проверка пользовательских сценариев;
- тестирование UI и логики приложения.

### Баг-репорты
По результатам тестирования оформлены баг-репорты с описанием:
- шагов воспроизведения;
- ожидаемого результата;
- фактического результата;
- приоритета дефекта.

### Отчёт о тестировании
Подготовлен итоговый отчёт по результатам тестирования с оценкой готовности приложения к релизу.
